# KYO 要件定義

## 目的
日々の生活を日次で記録し、健康状態可視化。
AIによる日次アドバイスと相談チャットでセルフケアを支援する

## 対象ユーザー
- 健康を意識している人で日々記録し、傾向を把握したいユーザー。
- 気にしている箇所(お肌や体の疲れ、体だるさ、風邪など)をよくするアドバイスをしてほしいユーザー

## 制約事項
- 医療的な診断は行わないこと。セルフケアの参考情報を目的とする。
- 日次アドバイス、相談チャットも医療的な行為は禁止。
- 注意文言として、主治医や皮膚科へ相談する旨を追加すること。

## 機能要件
  ### 利用目的の選択
    - ダイエットをゆるく意識したい
      - 狙い:食事と運動のバランスをみて、優しくアドバイスする。
        - ダイエットを我慢せず、帳尻で考える手法
    - 体調・健康を意識したい
      - 狙い：体の体調や健康を意識したい
        - 飲み過ぎなどの疲れや仕事の疲れなどに対して優しくアドバイスする。
        - 体の不調よりのユーザーを安心させる
    - 日々の健康を振り返りしたい(有料コース)
      - 狙い：日々の状態を詳しくみて丁寧にアドバイスをする
        - 健康への意識高いひと
  ### 記録を入力・編集
    - 睡眠時間(何時間)
    - 食事内容(フリー入力、テンプレ[朝: / 昼: / 夜: ])
    - ストレス度(0-10)
    - 今日の体調(フリー入力)
    - 運動(フリー入力)を記録できること
    - 食事内容からkcal・糖質・塩分の推定値を表示できること
    - 推定値はユーザーが上書きできること
  ### 日次アドバイスを生成
  ### 相談チャットで簡易応答
    - AI送信が未設定/失敗時はダミー/既定値で動作確認できること
  ### サブスク実装
    - 無料での利用は1日間記録、最大14日とする

## UI要件
- モバイルファーストで操作しやすいUIであること
- カレンダーで過去も入力・編集ができること
- ダッシュボード機能
  - 日々の記録したものをウィジットで表示
  - 日次アドバイスを生成
- 相談チャット/日々の記録をタブで切り替え 

## 非機能要件
- セッション内メモリのみで動作し、永続化しないこと
- 画面操作が軽快で、入力が簡単であること
