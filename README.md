# KYO 要件定義

### Why（目的）
- 日々の生活を日次で記録し、健康状態を可視化する。
- AIによる日次アドバイスと相談チャットでセルフケアを支援する。

### Who（対象ユーザー）
- 健康を意識して日々記録し、傾向を把握したいユーザー。
- お肌や疲れ、だるさ、風邪など気になる点へのアドバイスが欲しいユーザー。

### What（提供機能・画面）
- 利用目的の選択。
  - ダイエットをゆるく意識したい。
    - 食事と運動のバランスを見て、優しくアドバイスする。
    - ダイエットを我慢せず、帳尻で考える。
  - 体調・健康を意識したい。
    - 飲み過ぎや仕事疲れなどに優しくアドバイスする。
    - 体の不調寄りのユーザーを安心させる。
  - 日々の健康を振り返りしたい（有料コース）。
    - 日々の状態を詳しく見て丁寧にアドバイスする。
    - 健康への意識が高い人向け。
- 記録の入力・編集（当日/過去）。
  - 日付を選択して当日の記録を入力・編集できること。
  - カレンダーで過去の記録を閲覧・編集できること。
  - 今日の概要と今日のサマリーの間にカレンダーを配置すること。
- 記録項目
  - 睡眠時間（何時間）
  - 食事内容（フリー入力、テンプレ[朝: / 昼: / 夜: ]）
  - ストレス度（0-5）
  - 水分の入力(ml)(オプション入力)
  - 今日の体調（フリー入力）
  - 運動（フリー入力）
- 食事内容からkcal・タンパク質・脂質の推定値を表示できること。
  - 食事内容の横にAI推定ボタンを設け、推定栄養素を表示する。
  - 食事内容の下に計算した推定栄養素を表示する。
- 推定値はユーザーが上書きできること。
- 日次アドバイスを生成できること。
  - 今日の概要ページにアドバイス欄と生成ボタンを設け、押下後に表示する。
  - 日次アドバイスは専用ページを設けず、今日の概要ページで確認する。
- 記録の入力画面にはアドバイス生成ボタンを置かない。
- 相談チャットで簡易応答ができること。
- ダッシュボード機能。
  - 日々の記録をウィジットで表示。
  - 日次アドバイスを生成。
- 今日の概要ページで日次記録/相談チャットをタブで切り替える。
- 設定ボタンを設け、利用目的とアップグレード項目を用意する。
  - 利用目的はいつでも変更できるようにする。
- ニックネームを設定・変更できること。

### When（利用タイミング）
- 日次で記録し、必要に応じて当日/過去分を更新する。

### Where（利用環境）
- モバイルファーストで操作しやすいUIであること。
- iPhone,Android,Webブラウザ対応

### How（制約・非機能）
- 医療的な診断は行わないこと。セルフケアの参考情報を目的とする。
- 日次アドバイス、相談チャットも医療的な行為は禁止。
- 注意文言として、主治医や皮膚科へ相談する旨を追加すること。
- AI送信が未設定/失敗時はダミー/既定値で動作確認できること。
- セッション内メモリのみで動作し、永続化しないこと。
- 画面操作が軽快で、入力が簡単であること。

### How much（利用条件）
- サブスク実装を想定。
- 無料での利用は1日間記録、最大14日まで。

## 対象外/前提
- デプロイ先、リリース時期、予算は本書の対象外とする。
